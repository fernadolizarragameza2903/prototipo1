# 5. Estructura del repositorio GitHub

## Que se debe presentar

La organizacion de carpetas y archivos del proyecto. Esto demuestra orden, buenas practicas y facilidad para revisar el prototipo.

## Buenas practicas en GitHub

- Usar nombres claros para archivos.
- Mantener el README actualizado.
- Hacer commits con mensajes descriptivos.
- No subir archivos innecesarios del sistema.
- Publicar el proyecto con GitHub Pages.
