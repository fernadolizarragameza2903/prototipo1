# 6. Seleccion tecnologica justificada

## HTML5

Se eligio HTML5 porque permite construir la estructura semantica del sistema: landing page, login, encuesta, dashboard, chatbot, panel docente y secciones informativas.

## CSS3

Se uso CSS3 para crear un diseno moderno, responsive y visualmente atractivo. Permite manejar colores, animaciones, tarjetas, grillas, efectos glassmorphism y adaptacion a celular, tablet y laptop.

## JavaScript

JavaScript permite que la maqueta sea interactiva sin backend. Se usa para la encuesta STEM, resultados dinamicos, chatbot IA simulado, dashboard, filtros de proyectos, panel docente y almacenamiento local.

## localStorage

Se usa `localStorage` para simular una base de datos en el navegador. Guarda respuestas, XP, perfil dominante, historial y progreso del estudiante.

## Canvas

Canvas se utiliza para generar particulas animadas de fondo, reforzando el estilo tecnologico y futurista del prototipo.

## GitHub

GitHub se usa como control de versiones y repositorio publico del proyecto. Permite demostrar orden, historial de cambios y colaboracion.

## GitHub Pages

GitHub Pages se propone como medio de despliegue gratuito para publicar la maqueta web sin servidor ni configuracion avanzada.

## Justificacion general

Estas tecnologias fueron elegidas porque son suficientes para una maqueta funcional, visual y demostrable. No se requiere backend real ni IA real en esta etapa, ya que el objetivo es validar la experiencia, la interfaz y la idea del servicio.
